# 🥗 delicious_is_1000Kcal Tracker
OCR 기반 카카오톡 커뮤니티 데이터 자동 수집 및 분석 시스템

운동/식단 인증 모임의 수동 기록 업무를 자동화하기 위해 개발된 데이터 파이프라인 프로젝트입니다.
카카오톡의 텍스트 데이터를 OCR로 추출하고, Google Sheets API를 통해 실시간 통계 및 주/월간 대시보드를 구축했습니다.

## 🏗 System Architecture
```
[Mobile] KakaoTalk Capture
    ↓ (Google Drive Sync)
[PC] Input Directory (YYMMDD.jpg)
    ↓ (Java App - Swing UI / Scheduled Task)
[OCR Engine] Tesseract (Extract Text)
    ↓ (Keyword Parsing & Validation)
[Data Store] Google Sheets (Raw Data)
    ↓ (Auto-Calculation Logic)
[Dashboard] Weekly/Monthly Statistics
```

## 🛠 Engineering Decisions (핵심 고민과 선택)

### 1. 데이터 수집: OCR vs 크롤링
카카오톡의 폐쇄적인 API 정책상 직접적인 크롤링이 불가능했습니다.
대안으로 Tesseract OCR을 선택했으며, 인식률을 높이기 위해 특정 키워드(`운성`, `식성` 등) 기반의 파싱 로직을 직접 설계하여 데이터 정확도를 확보했습니다.

### 2. 인프라: Google Sheets API vs 자체 DB/웹
초기에는 웹 서비스 구축을 고려했으나, 모임 멤버들의 접근성과 별도의 앱 설치 없는 실시간 공유를 위해 Google Sheets를 데이터 저장소 겸 대시보드로 활용했습니다.
이를 통해 서버 비용 제로화와 협업 효율을 극대화했습니다.

### 3. 운영 효율: Jar 외부에 설정 파일 분리
스케줄러 주기나 멤버 명단 변경 시 재빌드 및 재배포하는 비효율을 방지하기 위해,
`config.properties` 와 `members.txt` 를 외부로 분리하여 런타임에 동적으로 로드하도록 설계했습니다.

### 4. 장애 대응: OCR 정정 루프 설계 (Feedback Loop)
OCR 인식 오류나 사용자 오타에 대응하기 위해, 구글 시트에서 수동 수정 후 특정 플래그(`Y`)를 입력하면
시스템이 이를 감지해 통계를 재계산하는 자기 수정형 로직을 구현했습니다.

### 5. 주간 통계 업데이트 주기
월요일에 한 번 확정하는 방식과 매일 업데이트하는 방식을 고민했습니다.
- **이번주현황 탭**: 매일 업데이트해서 이번 주 진행 상황을 실시간으로 확인
- **주간통계 탭**: 월요일에 지난주 확정 결과를 쌓아서 히스토리 관리

### 6. UI: 수동 실행 편의성
매번 터미널에서 실행하는 불편함을 해소하기 위해 Swing UI를 추가했습니다.
- 캡처 후 붙여넣기(Ctrl+V) → 자동 저장 + 실행
- 스케줄러와 UI 동시 실행

## 🔧 트러블슈팅

### OCR 인식률 최적화: 폰트보다 중요한 데이터 밀도
**문제**: 초기 테스트 시 특정 메시지의 한글 인식률이 낮아 데이터 정합성이 깨지는 현상 발생.

**가설 및 검증**:
- 폰트 문제: 맑은 고딕 → 나눔고딕 익스트라 → 굴림 으로 변경했으나 유의미한 개선 없음
- 크기 문제: 이미지 내 텍스트 크기를 100%에서 125%로 확대하여 재테스트

**결과**: 텍스트 크기를 키워 OCR 엔진이 분석할 수 있는 픽셀 정보량을 늘림으로써 인식률을 95% 이상으로 개선.

**배운 점**: 기술(Tesseract) 자체의 한계보다 입력 데이터의 품질(전처리)이 결과에 더 큰 영향을 미친다는 것을 체감함.

## 📋 데이터 정합성 보장 규칙
OCR 특성상 발생할 수 있는 데이터 오염을 방지하기 위해 아래와 같은 우선순위를 적용합니다.

- **실패 우선 원칙**: 완료 키워드가 있더라도 실패 키워드(`운실`, `식실`)가 포함되면 ❌로 처리
- **수동 정정 우선**: 시스템 데이터보다 사용자가 시트에서 직접 수정한 데이터(수정여부 `Y`)를 최우선으로 반영

## 구글 시트 구조

### 원본기록 탭
| 열 | 설명 |
|---|---|
| A | 날짜 |
| B | 이름 |
| C | 운동 (✅/❌/😋) |
| D | 식단 (✅/❌/😋) |
| E | 완료여부 (수식 자동 계산) |
| F | 수정여부 (Y: 수정됨 / D: 재실행 완료 / P: 무시) |

### 탭 구조
| 탭 이름 | 설명 |
|---|---|
| `원본기록` | 매일 멤버별 운동/식단 완료 여부 기록 |
| `이번주현황` | 매일 업데이트, 이번 주 진행 상황 실시간 확인 |
| `주간통계` | 매주 월요일 자동 생성, 지난주 확정 결과 누적 |
| `월간통계` | 종료일 다음날 자동 생성, 주차별 치팅 보너스 반영 |

## 추가해야 하는 파일 목록
아래 파일들은 보안상의 이유로 git에 포함되어 있지 않습니다. 직접 생성해야 합니다.

| 파일 경로 | 설명 |
|---|---|
| `src/main/resources/config.properties` | 구글 시트 ID, 인증 파일명, 이미지 경로, 스케줄러 설정 |
| `src/main/resources/서비스계정키.json` | Google Cloud 서비스 계정 JSON 키 |
| `src/main/resources/members.txt` | 멤버 이름 목록 (한 줄에 한 명) |
| `src/main/resources/tessdata/kor.traineddata` | Tesseract 한국어 언어팩 |

## 이미지 파일 형식
캡처 이미지는 아래 형식으로 저장해야 합니다.

- 저장 경로: `config.properties`의 `image.path.prefix` 경로
- 파일명 형식: `YYMMDD.png` 또는 `YYMMDD.jpg` (예: `260414.jpg`)
- 처리 완료된 이미지는 자동으로 `done/` 폴더로 이동됩니다
- 이미지에는 댓글 목록 전체가 보여야 합니다

## 사용 방법

### 자동 실행 흐름
```
PC 켜짐 (시작프로그램 자동 실행)
    ↓
UI 창 + 스케줄러 백그라운드 동시 실행
    ↓
구글 드라이브 동기화 (핸드폰에서 올린 이미지 자동 동기화)
    ↓
매 정시마다 스케줄러 실행
    ↓
수정여부 Y 체크 → 있으면 통계 재실행
최근 7일치 이미지 체크 → 미업로드 날짜 자동 업로드
처리 완료된 이미지 → done/ 폴더로 자동 이동
이번주 현황 탭 자동 업데이트
    ↓
매주 월요일: 주간 통계 자동 생성
종료일 다음날: 월간 통계 자동 생성
```

### UI 수동 실행
1. 카카오톡에서 해당 날짜 공지 댓글 전체 캡처 (`Win+Shift+S`)
2. UI 창에서 날짜 입력 (선택, 비우면 어제 날짜 자동)
3. **▶ 실행** 버튼 클릭
4. 클립보드 이미지 자동 저장 + 파싱 + 업로드

### OCR 오류 수정 방법
1. 구글 시트 원본기록에서 잘못된 데이터 직접 수정
2. F열 수정여부에 `Y` 입력
3. 스케줄러가 감지 후 자동으로 통계 재실행
4. 완료 후 `D` 로 자동 변경
