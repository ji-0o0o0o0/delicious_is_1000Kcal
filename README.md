# 🥗 delicious_is_1000Kcal Tracker

## 프로젝트 배경
운동과 식단 관리를 함께하는 모임에서 매일 카카오톡 공지에 완료 여부를 댓글로 남기고 있었습니다.
<br> 한 분이 매일 수동으로 확인하고 정리하는 것같은데 더 편한 방법이 없을까라는 마음으로 시작하게 되었습니다.

누가 얼마나 성실하게 했는지, 매일 어떻게 했는지를 모임 멤버들이 함께 볼 수 있도록
웹보다 접근성이 좋은 구글 시트에 자동으로 기록하는 방식을 선택했습니다.

## 고민했던 것들

### 카카오톡 데이터를 어떻게 가져올까?
카카오톡은 공식 API를 제공하지 않아 댓글을 직접 읽어올 수 없었습니다.
선택지는 두 가지였습니다.
- **크롤링**: 카카오톡 정책상 불가능
- **OCR**: 캡처 이미지에서 텍스트를 추출하는 방식

결국 OCR 방식을 선택했고, Tesseract를 사용해 한글 인식을 구현했습니다.

### 통계를 어디서 볼까?
멤버들이 함께 볼 수 있어야 했습니다.
- **웹 서비스**: 서버 구축과 배포가 필요해 복잡함
- **구글 시트**: 별도 설치 없이 링크만 공유하면 누구나 접근 가능

접근성과 편의성을 고려해 구글 시트를 선택했습니다.

### 멤버 관리를 어떻게 할까?
멤버가 추가되거나 변경될 수 있어서 코드에 하드코딩하는 건 좋지 않다고 판단했습니다.
`members.txt` 파일로 분리해서 코드 수정 없이 멤버를 관리할 수 있도록 했습니다.

### 민감 정보를 어떻게 관리할까?
구글 시트 ID, API 키 등 민감 정보를
`config.properties` 파일로 분리하고 `.gitignore`에 추가해 git에 올라가지 않도록 처리했습니다.

### 설정 변경 시마다 재빌드해야 할까?
스케줄러 시간 등 설정을 바꿀 때마다 jar를 다시 빌드하는 건 불편했습니다.
jar 외부에 `config.properties` 와 `members.txt` 를 두고 우선적으로 읽도록 해서
재빌드 없이 설정을 변경할 수 있도록 했습니다.

### 캡처 이미지를 어떻게 PC로 가져올까?
핸드폰으로 캡처하면 더 선명하고 편리하지만 PC로 옮기는 과정이 번거로웠습니다.
구글 드라이브를 활용해 핸드폰에서 업로드하면 PC에 자동 동기화되도록 했습니다.

### 스케줄러를 어떻게 자동화할까?
Java 내부 스케줄러만 사용하면 PC 재시작 시 매번 수동으로 jar를 실행해야 했습니다.
Windows 작업 스케줄러로 PC 시작 시 jar를 자동 실행하고,
Java 내부 스케줄러가 매 정시마다 동작하는 방식을 선택했습니다.

### 주간 통계를 언제 보여줄까?
주간 통계를 월요일에 한 번만 확정해서 보여주는 방식과 매일 업데이트하는 방식을 고민했습니다.
- **이번주 현황 탭**: 매일 업데이트해서 이번 주 진행 상황을 실시간으로 확인
- **주간통계 탭**: 월요일에 지난주 확정 결과를 쌓아서 히스토리 관리

두 탭으로 나눠서 둘 다 활용할 수 있도록 했습니다.

## 기술 스택
- Java 17
- Tesseract OCR (tess4j 5.8.0)
- Google Sheets API
- Logback

## 구글 시트 구조
| 탭 이름 | 설명 |
|---|---|
| `원본기록` | 매일 멤버별 운동/식단 완료 여부 기록 |
| `이번주현황` | 매일 업데이트, 이번 주 진행 상황 실시간 확인 |
| `주간통계` | 매주 월요일 자동 생성, 지난주 확정 결과 누적 |
| `월간통계` | 매월 1일 자동 생성, 월간 달성률 및 MVP |

## 추가해야 하는 파일 목록
아래 파일들은 보안상의 이유로 git에 포함되어 있지 않습니다. 직접 생성해야 합니다.

| 파일 경로 | 설명 |
|---|---|
| `src/main/resources/config.properties` | 구글 시트 ID, 인증 파일명, 이미지 경로, 스케줄러 설정 |
| `src/main/resources/서비스계정키.json` | Google Cloud 서비스 계정 JSON 키 |
| `src/main/resources/members.txt` | 멤버 이름 목록 (한 줄에 한 명) |
| `src/main/resources/tessdata/kor.traineddata` | Tesseract 한국어 언어팩 |

## 이미지 파일 형식
캡처 이미지는 아래 형식으로 저장해야 합니다.

- 저장 경로: `config.properties`의 `image.path.prefix` 경로
- 파일명 형식: `YYMMDD.png` 또는 `YYMMDD.jpg` (예: `260414.jpg`)
- 처리 완료된 이미지는 자동으로 `done/` 폴더로 이동됩니다
- 이미지에는 댓글 목록 전체가 보여야 합니다

## 사용 방법

### 자동 실행 흐름
```
PC 켜짐
    ↓
Windows 작업 스케줄러가 jar 자동 실행
    ↓
구글 드라이브 동기화 (핸드폰에서 올린 이미지 자동 동기화)
    ↓
매 정시마다 스케줄러 실행
    ↓
최근 7일치 이미지 체크 → 미업로드 날짜 자동 업로드
처리 완료된 이미지 → done/ 폴더로 자동 이동
이번주 현황 탭 자동 업데이트
    ↓
매주 월요일: 주간 통계 자동 생성 + 이번주 현황 초기화
매월 1일: 월간 통계 자동 생성
```

### 이미지 업로드 방법
1. 카카오톡에서 해당 날짜 공지 댓글 전체 캡처
2. 구글 드라이브 앱으로 `260414.jpg` 형식으로 저장
3. PC에 자동 동기화 후 스케줄러가 알아서 업로드

### 수동 실행
날짜를 입력해서 수동으로 업로드할 수도 있습니다.
1. 프로그램 실행
2. 날짜 입력 (6자리: `260414` 또는 4자리: `0414` - 연도 자동 추가)
3. 구글 시트에서 결과 확인

## 파싱 규칙
카카오톡 댓글에서 아래 키워드를 감지하여 완료 여부를 판단합니다.

### 완료 키워드
| 키워드 | 판단 |
|---|---|
| `운`, `운동`, `운성`, `운동성공`, `운동 성공` 포함 | 운동 완료 ✅ |
| `식`, `식단`, `식성`, `식단성공`, `식단 성공` 포함 | 식단 완료 ✅ |

### 실패 키워드 (완료보다 우선 적용)
| 키워드 | 판단 |
|---|---|
| `운실`, `운동실패`, `운동 실패` 포함 | 운동 실패 ❌ |
| `식실`, `식단실패`, `식단 실패` 포함 | 식단 실패 ❌ |

> 실패 키워드가 포함된 경우 완료 키워드보다 우선 적용됩니다.
> 예: `식실운성` → 식단 ❌ 운동 ✅
