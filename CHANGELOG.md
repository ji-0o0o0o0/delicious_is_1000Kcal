## [v1.2.0] - 2026-04-18
### 추가
- `WeeklyCurrentUploader`: 이번주 현황 탭 매일 자동 업데이트
- 이미지 처리 완료 후 `done/` 폴더 자동 이동
- 날짜 입력 4자리 지원 (연도 자동 추가, 예: `0418` → `260418`)
- 구글 드라이브 연동으로 핸드폰 캡처 이미지 자동 동기화

### 변경
- 스케줄러 매 정시 실행 방식으로 변경 (`scheduler.interval.hours`)
- 이미지 없으면 수정여부 체크 후 스케줄러 종료

### 고민했던 것들
- **주간 통계 업데이트 주기**: 월요일 한 번 확정 vs 매일 업데이트 고민
  → `이번주현황` 탭(매일)과 `주간통계` 탭(월요일 확정) 두 탭으로 분리해서 해결
- **달성률 기준**: 실제 기록된 날 vs 7일 고정
  → 7일 고정으로 결정 (기준 명확, 치팅데이 반영)

## [v1.1.0] - 2026-04-17
### 추가
- `ConfigLoader`: config.properties 중앙 관리
- `SheetsService`: 공통 로직 분리 (연결/정렬/통계계산/헤더관리)
- `WeeklyStatsUploader`: 매주 월요일 주간 통계 자동 생성
- `MonthlyStatsUploader`: 매월 1일 월간 통계 자동 생성
- `Scheduler`: 매일 지정 시간 실행, 최근 7일치 이미지 자동 체크
- 공동 순위 처리
- 중복 통계 덮어쓰기
- 날짜 기준 정렬
- 원본기록 제목/컬럼명 자동 생성
- jpg 파일 형식 추가 지원

### 변경
- jar 외부 config.properties, members.txt 우선 로드 (재빌드 없이 설정 변경 가능)
- tessdata 경로 jar/IntelliJ 환경별 자동 감지

### 고민했던 것들
- **tessdata 경로**: jar 내부 포함 시 Tesseract가 압축된 상태로 읽지 못하는 문제
  → jar 빌드 시 외부 복사, IntelliJ 실행 시 내부 리소스 자동 감지
- **설정 파일 위치**: jar 내부 vs 외부
  → jar 실행 위치 기준 외부 파일 우선 로드로 재빌드 없이 수정 가능하도록
- **스케줄러 자동화**: Java 내부 스케줄러만 사용 시 PC 재시작마다 수동 실행 필요
  → Windows 작업 스케줄러 → 시작프로그램 방식으로 변경

## [v1.0.0] - 2026-04-15
### 추가
- Tesseract OCR로 카카오톡 캡처 이미지 한글 인식
- `members.txt`로 멤버 관리 (가나다순 정렬)
- 운동/식단 완료 여부 파싱 후 구글 시트 자동 업로드

- Logback으로 로그 파일 관리 (30일 보관)
- config.properties로 민감 정보 분리

### 고민했던 것들
- **카카오톡 데이터 수집**: 공식 API 없음 → OCR 방식 선택
- **통계 공유 방법**: 웹 서비스(서버 필요) vs 구글 시트(링크 공유)
  → 접근성 고려해 구글 시트 선택
- **멤버 관리**: 하드코딩 vs 파일 분리
  → members.txt로 분리해 코드 수정 없이 관리
- **민감 정보 관리**: config.properties로 분리 후 gitignore 처리
